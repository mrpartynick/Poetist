# Первое знакомство с контролем версий

В настоящее время существует множество программ, которые позволяют хранить изменения по конкретному файлу, при этом не создавая копий самих файлов, а лишь храня в себе только вносимые изменения. Одной из таких программ является программа Git.

Git - программа контроля версий, позволяющая работать с различными версиями файла и отслеживать эти изменения (текст написан своими словами).

А вот определение из интернета : Git - система управления версиями с распределенной архитектурой.

## Язык разметки markdown

Markdown - язык текстовой разметки. С помощью него можно редактировать текст, создавать заголовки, производить выделение текста и т. д.

Приведем примеры. 

Для того, чтобы выделить текст курсивом, необходимо поставить вначале и в конце текста символ * : *Этот текст написан курсивом*.

Для того, чтобы выделить текст полужирным, необходимо поставить в начале и в конце текста символы ** : **Этот текст выделен полужирным шрифтом**.

Помимо этого, мы можем создавать ненумерованные и нумерованные списки. Делается это следующим образом : чтобы создать ненумерованный список, необходимо перед текстом поставить символ *. Каждая строка ненумерованного списка начинается с символа *:

* первый элемент ненумерованного списка
* второй элемент ненумерованного списка
* третий элемент ненумерованного списка

Для того, чтобы создать нумерованный список, необходимо поставить цифру с точкой и сделать пробел для написания текста (1. Пункт первый ):

1. Первый элемент нумерованного списка
2. Второй элемент нумерованного списка
3. Третий элемент нумерованного списка

А теперь попробуем использовать зачеркнутый текст. Для этого необходимо поставить символ две тильды (~) перед и после текста. Пример : ~~Этот текст будет зачеркнут~~

## Команды Git и работа с таблицей

Для работы с программой Git существуют специальные команды. О них мы сейчас и поговорим. Ниже будет представлен список команд в табличном виде и пояснения к ним.

| Команда| Действие команды| Примеры|
|:------:|:------:|:-------:|
|git config user.name "имя пользователя"| Добавление нового пользователя (имя)| git config user.name Holand|
|git config user.email "адрес электронной почты"| Добавление адреса электронной почты пользователя| git config user.email holand.joy@example.com|
|git --version| Позволяет просмотреть версию git| git --version|
|git init| Инициирует программу в выбраной папке| git init|
|git status| Отображает состояние рабочего каталога (внесенные изменения тоже отображаются)| git status|
|git add "наименование файла"| Добавление измененных данных| git add task.md|
|git config -m "Комментарий"| Сохранение внесенных изменений. Флаг -m позволяет нам добавить комментарий к сохранению (кавычки здесь нужно ставить обязательно)| git commit -m "Добавили некоторую информацию в файл"|
|git log| Отображает все изменения, внесенные за последнее время (все коммиты). Для более подробного отображения используется флаг -p| git log или git log -p|
|git diff| Просмотр изменений, внесенных в репозиторий| Выполняем команду git checkout для перехода в интересующий нас коммит и набираем git diff|
|git checkout| Позовляет перемещаться между коммитами и ветками| git checkot 022er8 или git checkout master|
|git branch| Создает новую ветку в проекте. Для перемещения между ветками необходимо ввести команду git branch "имя ветки, на которую хотим перейти"| git branch icecream|
|git branch -d "имя ветки"| Удаляет заданную ветку| git branch -d icecream|
|git merge "наименование ветки"| Объединяет две ветки в одну. Для объединения веток необходимо перейти в одну из них и указать вторую, которую мы хотим объединить с исходной| git merge icecream

Пояснение: при выполнении команды git merge может произойти конфликт (в том случае, если в обеих ветках в одном и том же месте (например, в строке №4) содержится какая-либо информация). Разрешить конфликт можно разными способами: сохранить оба изменения, затем отредактировать; оставить первый вариант; оставить второй вариант и. т. п.

## Работа с удаленными репозиториями

Для работы с удаленными репозиториями используется сервер Github. Чтобы начать работу, необходимо перейти по ссылке <https://github.com> и зарегистрироваться на сервере.

Github позовляет не только загружать свой репозиторий, но и использовать другие, находящиеся на сервере. Для этого нужно найти интересующий нас репозиторий, скопировать ссылку на него. Далее создаем папку на своем компьютере, в VSC переходим в нашу папку и в терминале набираем команду : git clone <скопированная ссылка>. В результате этих действий в нашей папке появится копия нужного нам репозитория. Для работы с ним необходимо выполнить команду cd <имя папки>, так как именно в ней инициализирован git.

Чтобы начать работу со своим репозиторием на сервере, необходимо локально создать папку, проект, внести изменения и сохранить. 

На сервере создаем репозиторий. Появятся варианты для работы с репозиторием. Выбираем нужный нам вариант (в данном случае через команды), набираем в терминале команды по нужному нам варианту. После этих действий наш проект появится на сервере в нашем созданном репозитории.

_Примечание_: при первой отправке после регистрации github попросит вас авторизоваться. Для этого необходимо следовать инструкциям, предоставляемым сервером во время процесса отправки.

## Отправка и обновление данных

Для того, чтобы отправить изменения на github, нужно выполнить следующее:

1. сохранить все изменения в своем проекте;
2. набрать в терминале команду git push.

Команда git push отправит наш репозиторий на github. 

Работать с проектом также можно на самом сервере.

Для того, чтобы загрузить изменения на локальный репозиторий, необходимо сделать следующее:

1. перейти в папку с репозиторием на своем компьютере;
2. в терминале набрать команду git pull.

Команда git pull загружает репозиторий (с изменениями) с сервера на компьютер.

## Pull request

Github предоставляет возможность предложить свои изменения по какому-либо проекту. Чтобы это сделать, необходимо проделать следующее:

1. на сервере найти интересующий нас проект;
2. нажать на кнопку fork;
3. прописываем наименование репозитория и нажимаем на кнопку create fork;
4. скопировать ссылку на репозиторий;
5. в терминале набрать команду git clone <ссылка>. Интересующий нас проект появится у нас на компьютере.

После этих действий:

1. Вносим изменения в проект и сохраняем;
2. Загружаем на сервер при помощи команды git push измененный репозиторий;
3. Заходим в репозиторий на сервере и переходим на вкладку pull request.

Далее нажимаем на кнопку pull request. Появится окно для отправки. Выбираем отправителя, пишем комментарий и текст для получателя и нажимаем на кнопку pull request.
